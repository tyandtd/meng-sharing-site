一次项目的关键决策

这页先作为项目复盘模板：每次做完一个小项目，就把目标、约束、选择和结果写下来，避免只记住结论而忘掉判断过程。

复盘结构

  • 目标：当时真正想解决什么问题。
  • 约束：时间、能力、工具、部署环境、维护成本。
  • 方案：为什么选这个实现，而不是另一个实现。
  • 结果：上线后是否达到预期。
  • 下次：哪些地方可以更快、更稳、更清楚。

本网站的一个选择

当前阶段选择纯 HTML、CSS 和 GitHub Pages，是为了先降低发布成本，把内容和访问地址跑通。等笔记数量增加，再考虑拆成 Markdown、模板或静态站点生成器。

后续补充

每次新增功能时，可以把关键决策补到这里。复盘不是写总结作文，而是给下一次行动留线索。
